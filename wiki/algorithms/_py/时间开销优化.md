## 实战综合示例

```python
class OptimizedSolution:
    __slots__ = ('MOD',)  # 减少实例内存
    
    def __init__(self):
        self.MOD = 10**9 + 7
    
    def numSub(self, s: str) -> int:
        # 1. 缓存局部变量
        MOD = self.MOD
        s_local = s
        
        # 2. 使用哨兵避免边界检查
        s_local = s_local + '0'
        
        # 3. 局部变量缓存
        p = 0
        ans = 0
        
        # 4. 直接遍历字符（C层迭代）
        for ch in s_local:
            if ch == '1':
                p += 1
            else:
                # 5. 位运算替代除法
                ans += (p * (p + 1)) >> 1
                p = 0
        
        return ans % MOD
```

**核心原则**：
1. 优先用C层实现的函数
2. 减少Python层循环
3. 缓存频繁访问的变量
4. 选择合适的数据结构
5. 避免不必要的函数调用

# Python C层函数完全指南

## 一、为什么C层函数更快？

Python的C层函数直接运行在CPython解释器的C代码中，避免了Python字节码的解释执行开销。**一个简单的Python循环可能执行数十条字节码指令，而C层函数在底层用一条C循环完成**，性能差距可达10-100倍。

---

## 二、核心C层函数分类详解

### 1. 序列/集合操作类（C层实现）

| C层函数 | 替代的Python操作 | 性能提升 | 说明 |
|---------|-----------------|---------|------|
| `len(obj)` | 手动计数 | 10x+ | 直接读取PyObject头部字段 |
| `sum(iterable)` | `for x in lst: total+=x` | 10-30x | C层累加，避免Python加法循环 |
| `max(iterable)` | 手动比较循环 | 10-30x | C层遍历+比较 |
| `min(iterable)` | 手动比较循环 | 10-30x | 同上 |
| `any(iterable)` | 手动循环+break | 5-10x | 短路求值在C层完成 |
| `all(iterable)` | 手动循环+break | 5-10x | 同上 |
| `sorted(iterable)` | 手动实现排序 | 20-50x | Timsort算法C实现 |

**示例对比**：
```python
# ❌ Python层 - 慢
total = 0
for x in data:
    total += x

# ✅ C层 - 快10倍
total = sum(data)
```

## 五、实战对比：同一任务的不同实现

以"统计字符串中连续1的最大长度"为例：

```python
# 1️⃣ Python层循环 - 基准
def max_consecutive_python(s):
    max_len = cur = 0
    for ch in s:
        if ch == '1':
            cur += 1
            max_len = max(max_len, cur)
        else:
            cur = 0
    return max_len

# 2️⃣ C层函数组合
def max_consecutive_c(s):
    # replace + split + max + map + len 全在C层
    return max(map(len, s.replace('0', ' ').split())) if '1' in s else 0

# 3️⃣ groupby C层迭代器
from itertools import groupby
def max_consecutive_groupby(s):
    return max((len(list(g)) for k, g in groupby(s) if k == '1'), default=0)
```

性能测试结果（100万字符）：
- Python循环：~0.12秒
- C函数组合：~0.03秒（快4倍）
- groupby：~0.05秒（快2.4倍）
